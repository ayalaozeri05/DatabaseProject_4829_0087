# דוח פרויקט מסד נתונים - שלב ה' (ממשק משתמש מתקדם)
## מערכת TransRoute Planner

ברוכים הבאים לשלב ה' של הפרויקט - פיתוח ממשק המשתמש (GUI).
בשלב זה בנינו ממשק Web מודרני, מקצועי ורספונסיבי המאפשר למשתמשי המערכת לנהל את מסד הנתונים של חברת התחבורה בקלות ובנוחות.

הפרויקט עומד ב-100% מהדרישות שהוגדרו לשלב ה', תוך שימת דגש רב על חוויית משתמש (UI/UX), ארכיטקטורה נקייה ושילוב עמוק של הלוגיקה העסקית (פרוצדורות משלב 4) ישירות בתוך התהליכים.

---

## 🏆 מענה מלא לדרישות הפרויקט (Checklist)

1. **מסך כניסה (Login Screen):** בכניסה לאפליקציה המשתמשים מופנים למסך התחברות ייעודי (`/login`), המגן על הגישה ומנתב את המשתמשים אל ה-Dashboard המרכזי רק לאחר מכן, ממנו ניתן לנווט לכל שאר מסכי המערכת.
2. **גישה לכל הטבלאות + פעולות CRUD מקיפות:** למערכת יש מסכים ייעודיים לכל הישויות המרכזיות (Routes, Trips, Vehicles, Drivers, Passengers, Stops, Registrations). לכל מסך יש יכולות מלאות של חילוץ (Select), הוספה (Insert), עדכון (Update) ומחיקה (Delete).
3. **תהליך עדכון חכם (Update):** כפי שנדרש, בעת לחיצה על "Edit", המשתמש מזין את המפתח (ID), והמערכת שולפת באופן אוטומטי ובזמן אמת את כל יתר שדות הרשומה לתוך חלון (Modal), משם המשתמש מבצע את העדכון.
4. **הסתרת מפתחות זרים (IDs):** בתצוגת הטבלאות, המשתמש לעולם לא רואה קודי ID (כמו `region_id` או `route_id`). במקום זאת, מבוצע `JOIN` בצד השרת כדי להציג את השמות עצמם (שם אזור, שם נהג, תיאור מסלול וכד').
5. **שילוב פונקציות ופרוצדורות (משלב 4) בתוך תהליכי העבודה:** הלוגיקה של בסיס הנתונים לא נשארה רק "על הנייר" – היא מהווה את מנוע האפליקציה:
   - **`schedule_new_trip`:** הוספת נסיעה חדשה ממסך ה-Trips משתמשת ישירות בפרוצדורה זו.
   - **`auto_assign_drivers_to_future_trips`:** שיבוץ נהגים מופעל אוטומטית ברקע ברגע שהמשתמש צופה ברשימת הנסיעות.
   - **`get_route_dashboard`:** רשימת המסלולים (Routes) מבוססת לחלוטין על RefCursor שמקורו בפונקציה זו משלב 4.
   - **`calculate_trip_occupancy`:** מסך הנסיעות מבצע `LATERAL JOIN` לפונקציה זו כדי להציג את אחוז התפוסה והסטטוס (FULL / AVAILABLE) בזמן אמת לכל נסיעה.
6. **מסך הרצת שאילתות ופרוצדורות מרוכז (Queries):** מעבר לשילוב העמוק בתהליכים, בנינו מסך ייעודי (`/queries`) המאפשר להריץ בלחיצת כפתור מעל ל-2 שאילתות משלב ב' ולמעלה מ-2 פרוצדורות משלב ד', ולהציג את התוצאות בטבלאות דינמיות.
7. **תצוגה ידידותית למשתמש (UI):** האפליקציה מעוצבת באופן מודרני (Glassmorphism, CSS Variables, Toasts), עם חלונות קופצים (Modals) לפעולות המהירות ביותר ללא טעינת דפים מיותרת.

---

## 🛠️ כלים וטכנולוגיות

* **צד שרת (Backend):** 
  * `Python` + `Flask` - ניהול הבקשות, הראוטינג ובניית ה-API.
  * `psycopg2` - דרייבר עוצמתי ויציב לניהול התקשורת מול בסיס הנתונים (ביצוע שאילתות, הרצת פונקציות וקריאת cursors).
* **מסד נתונים (Database):** `PostgreSQL` מריץ את הטבלאות, הטריגרים, הפרוצדורות והפונקציות שלנו בצורה יעילה.
* **צד לקוח (Frontend):** 
  * `HTML5` / `Vanilla CSS` - עיצוב מודרני מרהיב, Grid & Flexbox layouts.
  * `Vanilla JavaScript` (AJAX/Fetch) - תקשורת אסינכרונית עם השרת לטעינת נתונים חלקה (SPA-feel) בלי ריענון הדף.

## 🚀 הוראות התקנה והפעלה

כדי להפעיל את המערכת בסביבה המקומית, יש לבצע את השלבים הבאים:

1. **הכנת הסביבה והתקנת תלויות:**  
   פתח טרמינל (CMD או PowerShell) בתיקיית `phase5`, והתקן את ספריות הפייתון הדרושות בעזרת הפקודה:
   ```bash
   pip install -r requirements.txt
   ```
   *(וודא ששרת ה-PostgreSQL שלך דולק למטה ופרטי ההתחברות מעודכנים במידת הצורך בקובץ החיבור `db_connection.py`).*

2. **הפעלת השרת (App):**  
   באותו הטרמינל, הפעל את הקובץ הראשי:
   ```bash
   python app.py
   ```
   המסוף יציג הודעה שהשרת רץ בכתובת המקומית.

3. **כניסה לממשק הגרפי:**  
   פתח דפדפן (מומלץ כרום) וגלוש לכתובת:
   [http://localhost:5000](http://localhost:5000)  
   התחבר במסך ה-Login עם שם משתמש וסיסמה לבחירתך, ותועבר ל-Dashboard המרכזי.

## 📸 תמונות מסך של הפעלת האפליקציה

הקפדנו לתעד כל שלב באפליקציה. בתיקיית `photo` המצורפת תוכלו למצוא צילומי מסך המדגימים את שלל היכולות:
* מסך ההתחברות (Login)
* עמוד הבית המרכזי (Dashboard)
* מסך ניהול נסיעות (Trips) וחישוב תפוסה דינמי בעזרת פונקציות SQL
* חלונות יצירה ועדכון מבוססי מפתח (Modals) - חילוץ נתונים לצורך עריכה מהירה.
* הפעלת פרוצדורות (Auto Assign, Schedule) דרך מסכי המערכת (Queries / Trips).
